# Product Space v3

Прототип AI-воркспейса для продактов. Vite + React (plain JSX).

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
```

```bash
npm run build    # сборка в dist/
npm run preview  # превью прод-сборки
```

## Что внутри

- **2 персоны** (переключатель в топбаре):
  - **Продакт** — личное пространство: главная, библиотека навыков/промтов, артефакты, проекты, чат с AI.
  - **Owner** — обзор команды (статусы по каждому PM + портфель продуктов), плюс свой чат и библиотека.
- **Библиотека** — 20 навыков (multi-step воркфлоу → артефакт) + 20 промтов (one-shot), фильтры по этапу/типу, поиск. Общая для обеих персон.
- **Чат** — добавляешь навык/промт через «+», он «заряжается» как chip, на отправке собирается артефакт по структуре (PRD / JTBD / гипотезы и т.д.).

## Структура

```
src/
  App.jsx            роутинг персон/вью, состояние чата
  Sidebar.jsx        навигация (разная для персон)
  Chat.jsx           чат + пустой стейт + артефакт-карточки
  ui.jsx             примитивы (иконки с фолбэком, карточки, бейджи…)
  data.js            навыки, промты, проекты, PM-команда, шаблоны артефактов
  views/PMViews.jsx     дашборд / библиотека / проекты / артефакты
  views/OwnerViews.jsx  обзор команды / детальная по PM
  styles.css         дизайн-система (токены из v1)
```

## Заметки

- Артефакты генерируются из шаблонов (`docTemplate` в `data.js`) — это визуальный прототип, без реальных AID-вызовов.
- Кнопки «Копировать / В проект / Поделиться» в артефактах — визуальные заглушки.
- Иконки грузятся через namespace-import lucide с фолбэком на `Square`, поэтому отсутствующие имена не ломают сборку (бандл из-за этого крупнее — для прод-версии стоит перейти на точечные импорты).
